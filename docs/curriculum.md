## Difficulty Calibration

**Target range:** 12-year-old beginner to 25-year-old CS student. Both must feel challenged, never patronized.

**Three hint tiers (via SABLE):**

1. **No hints (default for experienced players):** SABLE only reports errors ("Line 4: NameError — 'threshhold' is not defined"). Player figures out the typo themselves.
2. **Leading questions (default for new players):** SABLE asks: "Your loop runs while battery is above 15. What happens when battery is exactly 15?" Guides without revealing.
3. **Show the fix (accessible via "SABLE, help me" command):** SABLE displays the corrected code with the changed line highlighted. Available after 3 failed attempts on any puzzle.

**Skip system:** Any challenge can be skipped. SABLE auto-generates a working but inefficient solution: robot uses 2x battery, circuit uses 3x power, model has 60% accuracy instead of 85%. The game continues, but the player's base runs at reduced capacity — a constant, gentle reminder that understanding matters. Skipped challenges remain in the journal and can be revisited.

**Age-adaptive difficulty signals:**
- A 12-year-old who nails T1 quickly gets less SABLE commentary (no hand-holding)
- A 25-year-old who struggles on circuits gets more Wren dialogue hinting at solutions
- Detection method: time-to-solve per puzzle. Below median = reduce hints. Above 2x median = increase hints. No explicit "difficulty select" screen.

## Anti-Patterns to Avoid

| Anti-Pattern | Why It Kills Fun | How AstroCode Avoids It |
|---|---|---|
| **Lecture before gameplay** ("First, let's learn about variables...") | Breaks immersion. Player came to play, not attend class. | Every concept is introduced by a crisis that already happened. The O2 is already dropping. The LED already exploded. Learning is the response, never the prerequisite. |
| **Disconnected exercises** ("Now practice: write 5 functions that return values") | Feels like homework. No stakes, no world impact. | Every line of code changes the game world. A function controls a real robot. A circuit powers a real system. Practice happens through play, not drills. |
| **Binary pass/fail grading** ("Wrong! Try again.") | Creates anxiety. Punishes experimentation. | Code runs and produces visible results — a robot walks into a wall, an LED burns out, a crop rots. Failure is diagnostic ("here's what happened"), never judgmental. SABLE never says "wrong." |
| **Forced linear progression** ("You must complete Lesson 3 before accessing Lesson 4") | Players who already know `if/else` are trapped in baby lessons. | Tier gates, not lesson gates. If a player can solve the Tier 2 gate challenge (Wren's purifier), they unlock T2 regardless of which T1 lessons they skipped. Competence is the key, not compliance. |
| **Abstract examples** ("Imagine a basket of apples...") | Players are on an alien planet, not in a classroom. | Every example uses in-game objects: sensor readings, robot batteries, atmospheric pressure, thaline purity. The "textbook" is the planet. |
| **Invisible progress** ("Great job! You learned loops!") | Patronizing. No tangible impact. | Every lesson ends with a visible, permanent change: a robot that works better, a base system that runs automatically, a new biome that becomes accessible. Progress is physical, not abstract. |
| **Sandbox without purpose** ("Here's a code editor. Do whatever you want!") | Freedom without motivation is paralysis, especially for beginners. | Freeform coding is available but never the introduction. Every concept arrives via a survival problem with clear stakes. Sandbox mode exists for replay and experimentation after the campaign. |

## Cross-Track Integration (5 Convergence Moments)

**1. Wren's Memorial Beacon (Day ~13, Brine Hollows)**
- **Programming:** Write an event callback that triggers the beacon at nightfall using a light sensor reading.
- **Circuits:** Wire a relay circuit — light sensor drives a transistor, transistor drives relay, relay switches the high-power beacon lamp.
- **Math:** Calculate resistor values so the transistor base current is sufficient to trigger the relay coil (Ohm's law + transistor gain).
- **Outcome:** A beacon that pulses on an alien hillside every night. Wren's voice cracks when she sees it.

**2. Solar Tracker System (Day ~17, Ferric Badlands)**
- **Programming:** Write a loop that reads the sun angle from a light sensor and updates the servo position every in-game hour.
- **Circuits:** Wire a servo motor to a microcontroller with PWM output. Add a voltage regulator to protect the servo from battery fluctuations.
- **Math:** Trigonometry for sun angle calculation; power equations to verify that the tracker's motor draw doesn't exceed the panel's output (net-positive energy check).
- **Outcome:** Solar panels that follow the sun. Power output triples. The base can run through the night on stored energy.

**3. Toxic Gas Early Warning Network (Day ~25, Spore Marshes)**
- **Programming:** Event-driven callbacks across a mesh network — Pips detect gas, broadcast alerts, robots auto-evacuate, base terminal logs the event.
- **Circuits:** Wire gas sensors into Pip drones. Build an op-amp to amplify weak readings from distant pockets. Add oscillator for mesh timing.
- **Math:** Exponential decay to calculate sensor detection range. Statistics (mean/variance) to distinguish real gas events from sensor noise.
- **Outcome:** Walk through the Spore Marshes without fear. Every toxic pocket mapped and avoided in real-time.

**4. Storm Prediction + Auto-Shelter (Day ~35, Resonance Peaks)**
- **Programming:** Regression model trained on atmospheric data. Prediction piped to an event system that commands all robots to shelter when storm ETA < 30 min.
- **Circuits:** EMP-hardened sensor stations (shielded breadboards with magnetite casings). Voltage regulator for stable readings during pre-storm EM fluctuations.
- **Math:** Regression math (minimize squared error). Probability (confidence intervals on predictions). Power budget for running the sensor array 24/7.
- **AI:** First real ML model deployment. The model's accuracy directly depends on how many sensors the player placed in earlier biomes and how well they preprocessed the data.
- **Outcome:** Zero EMP losses from Day 35 onward. The Peaks transform from death zone to resource-rich workplace.

**5. The Deep Signal Array (Day 55-60, Beacon Plateau)**
- **Programming:** Full pipeline: data collection, preprocessing, model training, real-time inference, frequency output to hardware. Mesh networking for live sensor feeds. Event-driven architecture for fault tolerance.
- **Circuits:** Piezoelectric transducer driven by a voltage-controlled oscillator. Op-amp for signal amplification. CPU (from logic gates) running on Atlas for local inference. Power routing from geothermal tap through voltage regulators to the Array.
- **Math:** Linear algebra (neural net forward pass). Optimization (loss minimization). Signal modulation math (frequency = f(model_output)). Power budget for the entire Array.
- **AI:** Neural network trained on 30+ days of resonance data. Architecture designed by player. Training loop tuned by player. The model must generalize to live, never-before-seen resonance patterns.
- **Outcome:** Everything the player built across 60 days converges in a single system. The signal fires. The game is won.

## Real-World Transferability

**Programming:** Players exit with genuine Python fluency — variables, functions, loops, data structures, classes, event-driven design, and API consumption. The syntax is 1:1 with real Python. A player who finishes AstroCode can open VS Code and write a web scraper, a data analysis script, or an automation tool without starting from zero.

**Electronics:** Players understand complete circuits (Ohm's law, voltage dividers, transistor switching), digital logic (NAND-to-CPU), and sensor integration. They can pick up an Arduino starter kit and wire a temperature-controlled fan on day one. The breadboard metaphor is identical to real-world prototyping.

**Math & Physics:** Players internalize that math is a tool for engineering decisions, not an abstract exercise. They can calculate power budgets, size resistors, evaluate signal decay, and interpret statistical measures. These are the exact calculations electrical engineers and data scientists perform daily.

**AI & Machine Learning:** Players understand the full ML pipeline: data collection, cleaning, train/test splits, model selection (trees vs. neural nets), training loops, loss curves, overfitting, and deployment. They can open a Jupyter notebook with scikit-learn or TensorFlow and recognize every concept. The visual node editor maps directly to how frameworks like Keras define architectures.
